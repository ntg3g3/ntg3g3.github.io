## Design Problem

This page's job is to show material practice, hand-making, form exploration, and interaction through objects. It should not read like a long UX case study.

The rebuild needs to distinguish three things visually:

- Final object beauty shots.
- Close detail/material shots.
- Use/interaction shots showing how the object is handled.

## Process Story

The available images emphasize finished objects and details more than step-by-step fabrication. The likely process story is: object exploration -> final form -> detail/texture -> interaction.

The best page story is therefore concise:

1. Handmade object studies.
2. Form and material details.
3. How the objects invite touch/play.

Do not add invented process diagrams, workshop steps, or design claims.

## Original Wix Page Logic

The original `/wood` page reads as a visual archive grid for craft work. It likely did not have a long written narrative. The contact sheet order suggests a mixed object gallery with repeated angles:

- It opens with the folded spiky/faceted object photos, including wide final views and close details. Because the object is metallic in appearance, this may be a cross-category artifact or a Wix asset-order issue.
- It then moves into the wooden toy/mechanism: wide shots, close views, overhead/angled views, and a final red-ball detail. These images work as a set and should be grouped together.
- It then shows the carved game-board object through detail closeups, fuller object views, an interaction shot with a hand, and colored balls in the carved basins. These should be grouped as a second major set.
- The logic is "object, detail, use", not "problem, process, solution".
- Text should be sparse: one sentence per object group at most, with image captions doing light identification.

For the rebuild, Woodcrafts should be mostly gallery-driven. Use large images for the carved board and the wooden toy, keep object groups clear, and avoid adding broad claims about design impact.
