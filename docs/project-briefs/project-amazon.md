## Original Wix Page Logic

The original page appears to follow a proposal-deck rhythm rather than a pure game gallery, but it should be framed as a playable VR/projection prototype rather than only a concept proposal. It starts with topic framing and research slides: "River" appears as the dominant setting, then "Summary" names the four learning themes. After that, the page moves into experience design artifacts: projection concept, layout board, screen flow, route map, and tree-route sketch. The public tutorial and playthrough videos should appear after these explanatory boards, because the boards teach the viewer how to read the demo.

The page logic is: first convince the viewer that the subject has educational value, then show how the subject becomes an embodied mixed-reality journey. Large images should be used for the river landscape, title screen, and layout-board because they establish setting and system. Smaller paired images should be used for route-map / tree-route-sketch and projection / screen-flow because they read as process comparisons.

The old layout likely worked because it let the diagrams carry most of the explanation. The rebuild should keep text short and let a few large boards explain the system.
