# Layout Plan: Archive Gallery Pages

Status: ready for implementation.

Scope:
- `product-design-intern`
- `soft-goods-project`
- `photography`
- `woodcrafts`
- `metalcrafts`
- likely also `affix`, `botto`, and `in-city-short-distance-ferry-terminal` unless later promoted.

## Intent

Archive pages should preserve old work visually without competing with current game/system design projects. They should feel like a clean image archive, not full case studies.

## Shared Rules

- Use a visual-first gallery rhythm.
- Keep text minimal: title, category/context, optionally one short sentence.
- Do not add process claims unless the original page clearly supports them.
- Do not force `My Impact`, long problem statements, or case-study framing.
- Images can be dense but should not be repeated.
- Captions should identify material, artifact, or stage only when helpful.
- Photography should have no explanatory paragraph beyond page title/context.
- Wood/metal/craft pages should group by material identity.
- Metal-looking folded objects belong in `metalcrafts`.

## Archive Index `/archive/`

Recommended:
- Reduce the intro to a short archive label and title.
- Remove the explanatory paragraph or compress it to one line.
- Let project cards carry the content.

## Detail Page Pattern

Hero:
- Title and category.
- Cover image.
- One short context line if needed.

Body:
- One gallery-style visual section.
- Optional second section only when the old page has a clearly different group, such as sketch/process/final.

Avoid:
- Long left-text/right-image case-study pattern.
- Repeating page-level summary text in each section.
- Overexplaining Photography.

## Project Notes

| Project | Direction |
| --- | --- |
| Product Design Intern | Gallery of internship/product artifacts; minimal role/context. |
| Soft Goods Project | Gallery of materials, prototypes, and final textile/soft-goods output. |
| Photography | Pure gallery. No explanation. |
| Woodcrafts | Gallery of wood objects and process/final images only. |
| Metalcrafts | Gallery of metal objects; ensure folded metal-looking work is placed here. |
| AFFIX | Keep as archive/product concept unless promoted later. |
| BOTTO | Keep visual and prototype-forward; short interaction context only. |
| In-City Short-distance Ferry Terminal | More design-process-heavy than craft archive; can stay as archive case study if images require it. |

## Acceptance Criteria

- Archive pages do not look more important than main game/work pages.
- The archive index has minimal prose.
- Photography has no explanatory body copy.
- Each archive detail page is image-led and readable on mobile.
- Build and image verification pass.
